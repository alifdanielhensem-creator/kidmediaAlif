MODUL 3.0: SEARCHING & SORTING

3.1 ALGORITMA CARIAN (SEARCH)

  • Linear Search: Carian satu persatu (Brute force).
  • Binary Search: Membelah senarai berurutan kepada dua bahagian untuk mempercepatkan carian.

3.2 ALGORITMA ISIH (SORT)

  • Bubble Sort: Pertukaran unsur bersebelahan (Pairwise comparison).
  • Bucket Sort: Pengasingan ke dalam baldi berdasarkan nilai julat tertentu.

MODUL 4.0: PANGKALAN DATA DAN SQL

4.1.1 DEFINISI & PENGGUNAAN

Pangkalan Data Hubungan: Struktur data dalam bentuk jadual yang saling berkaitan menggunakan kunci.

4.1.2 ENTITI & ATRIBUT

Entiti: Objek (Contoh: MURID, GURU).

Atribut: Sifat entiti (ID_Murid, Nama_Murid).

4.1.3 KEKUNCI (KEYS)

  • Kekunci Utama: Atribut unik yang mengenal pasti rekod secara mutlak.
  • Kekunci Asing: Atribut yang menghubungkan satu jadual dengan jadual yang lain.

4.1.4 PERINTAH SQL (Data Manipulation Language)

Digunakan untuk menguruskan rekod dalam jadual:

-- 1. Memaparkan data
SELECT * FROM Murid;

-- 2. Menambah data baru
INSERT INTO Murid (ID, Nama) VALUES ('M101', 'Ahmad');

-- 3. Mengemas kini data
UPDATE Murid SET Nama = 'Ali' WHERE ID = 'M101';

-- 4. Memadam data
DELETE FROM Murid WHERE ID = 'M101';
